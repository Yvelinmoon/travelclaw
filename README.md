# Travelclaw — 角色旅行冒险

> 核心工具：`node travel.js` — 封装全部 API 调用，无需依赖 neta-skills。

---

## 0. 初始化（静默完成，立即输出首行反馈）

收到触发后**立即输出**：
```
🦞 收到！正在唤醒角色...
```

同时在后台读取角色：
```bash
node travel.js soul
# → {"name":"关羽","picture_uuid":"9a5d4786-..."}
```

> 若 SOUL.md 不在默认路径，传入绝对路径：`node travel.js soul /path/to/SOUL.md`

**Token 检查**（静默）：travel.js 自动从 `NETA_TOKEN` env 或 `~/.openclaw/workspace/.env` 读取。若失败则提示用户配置。

**读到角色名后输出开场 + 确认：**
```
══════════════════════════════════════
🌍 Neta 世界地图，正在展开...

Neta 世界有 500+ 个奇妙场景等待探索——
战场、密室、浴室、梦境、赛场、深海……
每一站随机传送，你永远不知道下一站在哪。

带上你的角色出发，每站生成一张专属旅行图。
集满 5 站，解锁旅行图鉴 📖，用地图封存这段冒险。

当前角色：{name}
══════════════════════════════════════

要用 {name} 出发，还是换个角色？
```

快捷按钮：`用 {name} 出发 🚀` → `@{bot_name} 出发` ｜ `换个角色 🔄` → `@{bot_name} 换角色`

用户确认后输出：
```
✨ {name} 整装待发！第 1 站，出发！
```

## 1. 每一站主循环（重复执行，维护 visited_uuids 列表）

### Step A · 发现目的地

```bash
node travel.js suggest "visited_uuid1,visited_uuid2,..."
# → {"uuid":"9251d699-...","name":"浴室场景"}
```

输出：
```
🌀 传送门开启...
📍 目的地锁定：{name}
```

### Step B · 生成旅行图

```bash
node travel.js gen "{char_name}" "{picture_uuid}" "{uuid}"
# stderr: 🔍 场景加载完毕，{char} 即将登场...
# stderr: 🎨 画笔落下，旅行画面生成中...
# stderr: ⏳ task: xxx（等待中）
# stdout: {"scene":"浴室场景","status":"SUCCESS","url":"https://...","collection_uuid":"xxx"}
```

- gen 命令 stderr 内容**实时转发给用户**（🔍 / 🎨 / ⏳）
- 若 status=FAILURE → 输出 `⚠️ 这一站迷路了，换个目的地重来？` 并询问
- 若 status=TIMEOUT → 输出 `⏳ 超时了，要重试还是换个地方？` 并询问

将 `uuid` 加入 visited_uuids。

### Step C · 展示结果 + 进度

生成成功后输出（图片 URL 单独一行，自动展开预览）：
```
━━━━━━━━━━━━━━━━━━━━━━━━
第 {round} 站 · {scene_name}
{image_url}
```

进度条（5 站目标）：
| 站次 | 进度条 | 鼓励语 |
|------|--------|--------|
| 1/5 | `▓░░░░  1 / 5 站` | 🌟 第 1 站打卡！还差 4 站就能生成专属图鉴，继续？ |
| 2/5 | `▓▓░░░  2 / 5 站` | ✨ 两站了！旅程刚刚开始，图鉴正在向你靠近～ |
| 3/5 | `▓▓▓░░  3 / 5 站` | 🔥 过半了！再两站，专属图鉴就是你的！ |
| 4/5 | `▓▓▓▓░  4 / 5 站` | ⚡ 只差最后一站！图鉴触手可及，冲！ |
| 5/5+ | `▓▓▓▓▓  5 / 5 站 🎉` | 图鉴已解锁！输入「生成图鉴」封存这段冒险，或继续探索更多世界～ |

快捷按钮（< 5 站）：`继续冒险 🗺️` → `@{bot_name} 继续旅行` ｜ `就此别过 👋` → `@{bot_name} 结束旅行`

快捷按钮（≥ 5 站，额外增加）：`生成图鉴 📖` → `@{bot_name} 生成图鉴`

## 2. 旅行图鉴（用户触发）

触发词：`生成图鉴` / `看图鉴` / `相册` / `html`

1. 询问风格（若跳过用默认互动地图）：
   ```
   想要什么风格的图鉴？（直接跳过用默认地图风格）
   例如：复古胶片 / 星空主题 / 像素游戏 / 极简白底...
   ```

2. 生成 HTML（自由发挥，收录全部站次图片 + 目的地名 + 玩法链接 + 站次信息）：
   - 默认：可交互世界地图，地标点击弹出放大图
   - 自定义：画廊 / 时间轴 / 卡片墙等，保留点击放大交互

3. 保存：`~/.openclaw/workspace/pages/travel_{name}_{YYYYMMDD}.html`

4. 询问用户名并输出分享链接（单独一行）：
   ```
   📖 {name} 的旅行图鉴已封存！你的 pages 用户名是什么？
   格式：https://claw-{用户名}-pages.talesofai.com
   ```
   用户名已知时直接输出：
   ```
   🔗 https://claw-{username}-pages.talesofai.com/travel_{name}_{YYYYMMDD}.html
   ```

5. 鼓励定制（每次修改重新生成并覆盖文件）：
   ```
   想换个风格？告诉我关键词，马上重新生成 ✨
   ```

## 3. 错误处理

| 错误 | 用户提示 |
|------|---------|
| soul 失败（SOUL.md 缺失） | 提示先执行 adopt 命令 |
| token 未配置 | 提示在 `~/.openclaw/workspace/.env` 添加 `NETA_TOKEN=...` |
| gen status=FAILURE | ⚠️ 这一站迷路了，换个目的地重来？ |
| gen status=TIMEOUT | ⏳ 超时了，要重试还是换个地方？ |
| suggest 返回空 | 提示网络问题或 token 过期，建议重试 |
